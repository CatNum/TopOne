**名称（中文）**：Markdown 格式规范化与重点结构化

**描述（中文）**：在新增或修改 Markdown（.md）文档时，整理成统一结构，确保包含标题与可抓取的重点区域，提升阅读体验与重点抽取效率。

# md-formatting

## 何时使用
- 用户要创建一个新的 `*.md` 文档。
- 用户要更新/修改一个已有的 `*.md` 文档（即使只是小幅改动）。
- 用户希望 Markdown 输出遵循统一结构，从而更便于阅读与抓取重点。

## 核心承诺（这里“保持一致”的含义）
- 输出 Markdown 必须包含以下“固定区块”，并且每个区块都必须：
  - 只出现一次
  - 位于固定位置
  - `# 标题`（文档标题；位于全文第一个顶层标题位置）
  - `## 重点`（可快速抓取的要点列表；必须紧跟在 `# 标题` 块之后）
- 文档其余内容应按“内容需要”来组织：
  - 只有当文章确实需要进一步分段时，才新增或调整小标题。
  - 使用 `##` / `###` 来承载内容（不要强制固定成某一种深度策略，比如永远只用 `###`）。
  - 对“旧文”（用户已有的现成 Markdown）：不需要改动标题的层级深度（比如把 `##` 改成 `###` 等）；仅当用户明确要求调整结构时，才允许改动标题层级深度。
- 不引入新的事实/数字。尽量少改文字表述；主要聚焦在结构与格式规范化。

## 输出模板（最新结构）
1. `# <title>`
2. `## 重点`
3. 其余内容（按内容分段的段落与标题）

## 标题处理
1. 如果源文档已存在 `# ...` 标题：保留其含义，并用作文档标题。
2. 如果不存在 `# ...`：
   - 若前几行出现了明显“标题用途”的文本（例如 `## ...` / `### ...` 但语义明确是在当标题），则使用它。
   - 否则从文件名或文档首段推导标题。
3. 如果仍无法较为确定地推断标题：将标题设置为 `# 待确认标题`，并在 `## 重点` 下增加一句简短说明（示例 bullet：`- **待确认**：标题不确定，建议补充更明确的主题句`）。

## `## 重点` 规则（便于阅读与抓取）
1. `## 重点` 使用 bullet 列表呈现重点，不做固定数量限制。
2. 每条 bullet 要短且独立可读：
   - 以加粗的关键信号短语开头。
   - 后接 1-2 句说明这条重点的要点/结论。
3. bullet 内容必须来自源文档文本（不凭空编造新事实/数字）。
4. 如果文档偏技术/概念性较强，至少包含 1 条同时覆盖：
   - 核心概念
   - 若文档有流程/步骤：流程或步骤要点
   - 若文档有约束/前提：关键约束或假设

## 标题结构规则（内容驱动，不强制深度）
1. 用标题分割“语义一致”的内容块：
   - 更倾向用 `##` 承载主要段落/模块
   - 用 `###` 承载主要模块下的子段
2. 只有在内容确实需要更细粒度承载时，才进一步加深层级（例如：不同部分明显割裂、定义与例子需要区分、流程与坑点需要分开）。
3. 避免生成空标题段落：若某段没有实质内容，则合并到最近的上层标题。
4. 保持顺序合理：
   - 从概览/目标开始
   - 再展开细节
   - 若源文档有收尾/总结，则放在最后

## Mermaid 图（文本说明 + mermaid 代码块）
- 如果文档包含“关系/流程/工作流”，且用图能显著提升理解：
  1. 先补一句短文本，说明该图表达的含义/展示目的。
  2. 再给出一个 ` ```mermaid ` 代码块。
- 不要过度使用图；只在“扫读时更易理解”时才添加。

## 代码块与表格
- 若源文档已存在代码块或表格：
  - 除非明显损坏/缺失必要的最小格式，否则尽量保留原样。
- 不强制要求代码块必须带特定语言标签（例如 ` ```go ` 与未标注都允许）。
- 表格也不强制统一对齐风格，只保证最基本的正确性与可读性。

## 旧文更新规则（重要）
- 即使用户只对已有 `md` 文档做了很小的改动，你也要把整篇文档按“上述最新结构规则”对齐（重点确保 `# 标题` 与 `## 重点` 存在且已更新）。
- 同时保持改动最小化：
  - 主要做区块补齐与轻量重排，但保持原有标题层级深度不变（除非用户明确要求调整结构）
  - 更新 `重点` 以反映本次新增/修改的内容
  - 避免对正文段落逐字重写

## 输出前自检
- [ ] 文档开头是“且仅有一个”顶层 `# ...` 标题。
- [ ] 文档其他位置不存在额外的顶层 `# ...` 标题。
- [ ] `## 重点` 只出现一次，并且紧跟在 `# 标题` 块之后（中间不插入 `##/###` 标题块）。
- [ ] `## 重点` 使用 bullets，且每条 bullet 以 `- **`（加粗关键信号短语）开头。
- [ ] `## 重点` 的每条内容都能在源文档中找到依据（不新增虚构事实/数字）。
- [ ] 其余标题均为“内容驱动”（确有需要才使用 `##/###`），且没有空标题段落。
- [ ] 如果包含 Mermaid 图：遵循“先一句文本说明，再输出 ` ```mermaid ` 代码块”的模式。

## 输出契约
- 只输出最终的 Markdown 内容。
- 不要在 Markdown 之外附带说明文字。
