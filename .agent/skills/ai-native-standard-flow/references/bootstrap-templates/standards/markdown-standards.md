# Markdown Standards

> 模板提示：本文仅供参考，请结合项目实际进行人工完善与确认。

## 标题层级规范

- `#`（H1）：每个文件仅一个，用于文件标题。
- `##`（H2）：主章节，用于一级分类（如"背景"、"目标"、"决策"）。
- `###`（H3）：子章节，用于二级细分，不超过三层嵌套。
- 禁止跳级使用（如从 H2 直接跳到 H4）。

## Mermaid 使用规则

- 流程图统一使用 `flowchart TD`（自上而下）或 `flowchart LR`（左右），不混用。
- 状态图使用 `stateDiagram-v2`；序列图使用 `sequenceDiagram`。
- 图形节点命名使用英文，注释可用中文。
- 每个 Mermaid 图须有前置说明文字，不允许图形孤立存在。

## 必填字段规则

所有协作文档（需求、设计、决策）必须包含以下区块，缺失视为文档未完成：

| 字段 | 适用文档 | 说明 |
|------|---------|------|
| **决策** | 设计文档、ADR | 明确写出选定方案及理由 |
| **风险** | 设计文档、决策文档 | 列出已识别风险与缓解措施 |
| **下一步** | 所有协作文档 | 明确后续行动项与负责人 |

## 其他规范

- 表格列宽保持统一，单元格内容避免换行。
- 代码块须标注语言类型（如 ` ```js ` 、` ```bash `）。
- 外部链接须确保可访问，内部引用使用相对路径。
