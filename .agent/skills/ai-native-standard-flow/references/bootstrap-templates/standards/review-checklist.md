# Review Checklist

> 模板提示：本文仅供参考，请结合项目实际进行人工完善与确认。

## 正确性

- [ ] 是否满足需求文档中的所有验收标准？
- [ ] 是否覆盖主流程和异常分支？
- [ ] 边界条件（空值、最大值、并发）是否已测试？

## 安全

- [ ] 是否存在未校验的用户输入（XSS、SQL 注入、命令注入）？
- [ ] 敏感信息（密钥、Token、PII）是否避免明文记录或暴露？
- [ ] 权限校验是否在服务端执行，而非仅靠前端控制？

## 性能

- [ ] 是否存在 N+1 查询或不必要的循环？
- [ ] 新增接口的响应时间是否在基线以内？
- [ ] 缓存策略是否合理，是否存在缓存穿透/击穿风险？

## 可维护性

- [ ] 命名是否清晰，逻辑是否自解释？
- [ ] 是否引入了不必要的全局状态或副作用？
- [ ] 公共工具/组件是否有单测覆盖？

## 可回滚性

- [ ] 数据库变更是否可回滚（是否有 down migration）？
- [ ] Feature Flag 是否就位，可以快速关闭新功能？
- [ ] 本次变更的回滚步骤是否在 PR 描述中说明？
