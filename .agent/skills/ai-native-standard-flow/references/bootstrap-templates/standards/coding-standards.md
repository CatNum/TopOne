# Coding Standards

> 模板提示：本文仅供参考，请结合项目实际进行人工完善与确认。

## 通用规范

### 命名

- 变量、函数使用 `camelCase`；类/类型使用 `PascalCase`；常量使用 `UPPER_SNAKE_CASE`。
- 名称须表达意图，避免单字母变量（循环索引除外）。
- 布尔变量以 `is`/`has`/`can` 开头。

### 异常处理

- 不允许静默吞掉异常（空 catch 块须附注释说明原因）。
- 对外接口的错误必须返回结构化错误对象，包含 `code`、`message`。
- 业务异常与系统异常分类处理，不混用。

### 注释原则

- 注释解释"为什么"，而非"是什么"——代码本身应自解释。
- 复杂算法、非直觉性决策必须附注释。
- 废弃代码删除而非注释掉，必要时在 ADR 或 PR 中留痕。
