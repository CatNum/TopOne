# Design

> 模板提示：本目录按**产品发布版本**组织，每个版本独立存放。
> 当前目录为版本索引，具体技术方案文档在版本子目录中维护。
>
> **定位**：本目录只记录“系统如何实现”——架构、模块边界、接口契约、数据流、状态管理、存储方案、技术权衡与风险。
> **不得包含**：PRD 正文、页面结构说明、按钮跳转、交互文案、纯文字原型描述。这些内容应写入 `docs/requirements/`。

## 新版本创建指引

每次开始新产品版本的技术方案工作时：

1. 创建新版本目录：`mkdir -p docs/design/vY.Y`
2. 在本索引表中登记版本状态
3. 在新版本目录的 `README.md` 中记录该版本的技术方案
4. 需求背景、页面规则、交互流程请引用 `docs/requirements/README.md`，不要重复抄写
