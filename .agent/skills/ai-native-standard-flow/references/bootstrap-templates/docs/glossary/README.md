# Glossary

> 定位：本文是项目的**活字典**，永远反映当前可用的业务术语。
> 不按版本分区——术语演变通过"曾用名"和"变更原因"字段在同一行内记录。

## 术语表

| 术语 | 定义 | 业务场景 | 曾用名 | 变更原因 | 引入版本 | 来源/负责人 |
|------|------|---------|--------|---------|---------|-----------|

## 维护规则

- **新增术语**：直接追加一行，填写全部字段；在 PR 描述中注明"新增术语：XXX"。
- **术语重命名**：不修改原行，在原行的"曾用名"字段填入旧名，"变更原因"说明为什么替换。
- **术语废弃**：不删除该行，在"术语"列用删除线标注（`~~旧术语~~`），"定义"列改为"已废弃，现用：XXX"，"变更原因"说明废弃理由。
- **定义修订**：直接更新"定义"列，git commit 记录修订历史。
