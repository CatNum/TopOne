# Decisions

> 模板提示：本文仅供参考，请结合项目实际进行人工完善与确认。

记录关键决策、备选方案和取舍依据，保证可追溯。

## 目录职责

`docs/decisions/` 采用“根目录索引 + 版本目录承载 ADR”的结构。

规则：
- 根目录 `README.md` 只说明规则与维护版本索引。
- 每个产品版本在 `docs/decisions/<version>/` 下维护该版本 ADR。
- ADR 文件名格式：`ADR-NNN-简短标题.md`。
- ADR 内部应注明状态，适用版本可作为补充信息保留。
- 已有 ADR 原则上不修改；如决策被推翻，新建 ADR 并引用原条目。
