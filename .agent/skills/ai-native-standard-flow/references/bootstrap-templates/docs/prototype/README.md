# Prototype

> 模板提示：本目录按**产品发布版本**组织，每个版本对应一份交付前的原型快照。
> 版本发布后，该版本原型成为只读历史证据，不再修改。
>
> **定位**：本目录只存放高保真或可交付的可视化原型证据，例如 Figma 链接、页面截图、交互录屏、页面清单。
> **不得包含**：纯文字页面说明、按钮跳转、业务规则、需求正文；这些内容应写入 `docs/requirements/`。

## 目录结构说明

每个版本子目录的推荐结构：

```
docs/prototype/
└── v1.0/
    ├── README.md          # 原型说明、页面清单、交互资产入口
    ├── screens/           # 各页面截图（PNG），命名格式：序号-页面名
    │   ├── 01-login.png
    │   ├── 02-dashboard.png
    │   └── 03-checkout.png
    └── flows/             # 关键交互流程录制（GIF），命名格式：序号-流程名
        ├── 01-register-flow.gif
        └── 02-payment-flow.gif
```

**截图命名规则**：`序号-页面名.png`，序号按页面在主流程中的顺序排列，便于浏览时还原用户路径。

**录屏建议**：时长控制在 10 秒以内，只录关键交互步骤；用 LICEcap / Kap / ScreenToGif 等工具录制。

## 新版本创建指引

1. 创建新目录：`mkdir -p docs/prototype/vY.Y/{screens,flows}`
2. 复制下方版本目录模板到 `vY.Y/README.md`
3. 在索引中登记原型链接、保真度与状态
4. 若当前只有文字需求，还没有高保真原型，不要把需求说明写进本目录，应先维护在 `docs/requirements/`
