# 简历与面试素材

## 项目概览
- 项目名称：`transcript-refine`
- 业务目标：将长视频课程转写稿变成可直接用于首次学习的结构化精炼稿。
- 技术栈：Markdown、Mermaid、规则引擎式写作约束、证据源增强流程。
- 我的角色：流程与规范设计者，负责“质量边界 + 可追溯输出”落地。

## 关键成果（STAR）
### 成果 1：搭建转写精炼主流程
- Situation（场景）：转写稿存在大量口语噪声、重复与术语错误。
- Task（任务）：建立可重复执行的精炼流程，保证知识完整性。
- Action（行动）：定义保序、纠错、去重、案例保留、教学闭环规则。
- Result（结果，尽量量化）：形成可稳定复用的学习稿生成基线。

### 成果 2：引入证据源增强机制
- Situation（场景）：仅凭转写稿存在术语误听与参数失真风险。
- Task（任务）：提升纠错准确性与可追溯性。
- Action（行动）：将 PDF/TXT/代码扫描纳入主流程，统一来源标注格式。
- Result（结果，尽量量化）：关键术语与参数纠错可靠性提升，结论可回溯。

## 可量化指标
- 准确性：来源标注覆盖率提升，纠错证据链更完整。
- 效率：学习者对原视频回看的依赖下降。
