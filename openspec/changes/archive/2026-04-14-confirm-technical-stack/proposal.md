# Proposal

## 背景

TopOne 已完成产品方向、需求、交互原型与 UI 原则确认。当前阶段需要从“产品定义”进入“技术栈确认”，把后续开发所依赖的客户端栈、本地数据层、同步策略与工程基线正式冻结。

目前仓库已经具备 AI Native 协作、文档、合规与流程基线，但尚未建立应用实现层面的技术栈基线。特别是以下决策仍未正式冻结：
- D1：技术栈选型策略
- D2：跨端交付路径
- D3：数据存储与同步策略

同时，`Lint / Type Check / Unit Test` 在当前合规检查中仍属于未确认状态，说明工程基线也必须在本次变更中一并确定。

## 问题

如果继续在没有正式技术栈确认的情况下推进开发，会出现以下问题：
- 平台实现路径反复摇摆
- `macOS` 常驻展示层 / 状态栏 / 主窗口能力难以统一收敛
- 本地存储与后续同步边界不清晰
- 工程基线无法落地，导致后续开发难以形成稳定节奏

## 本次变更目标

本次 change 只负责确认以下内容：
- Apple-first MVP 客户端技术栈
- 本地数据层方向
- MVP 同步策略与后续同步路径
- lint / build / test / CI 工程基线
- deferred 范围

## 非目标

本次 change 不进入以下内容：
- 具体业务功能开发
- Android / Windows 运行时落地方案
- 自建后端设计
- 页面级实现细节
- 最终视觉与高保真交互打磨

## 预期结果

变更完成后，项目应形成一套可执行、可验证、可回写文档的技术基线，使后续工作可以基于 OpenSpec 任务推进，而不是继续通过临时讨论决定方向。
