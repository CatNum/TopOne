## Why

当前需求文档与原型已经明确任务页是 TopOne v1.0 的核心管理入口。此前的目标基础 change 已废弃，下一步需要以任务页为实际动态程序入口，落地长期任务、日常任务、添加、编辑、删除、进度管理、当前专注锁定与提前切换理由等完整闭环。原型仅作为信息架构、视觉层级与关键交互参考，不能实现成一比一静态页面。

## What Changes

- 建立任务页所需的长期任务与日常任务领域模型、持久化字段与基础服务。
- 实现长期任务列表：当前专注任务、暂停的任务库、已达成的里程碑，并补充任务页顶部精神性导语作为页面第一段引导。
- 实现日常任务列表：日常任务可绑定到任意长期任务，用户可编辑和删除任意日常任务，但只有当前 TopOne 下的日常任务可更新状态。
- 支持添加、编辑、删除长期任务与日常任务，并遵守数量、日常任务状态、等级与归属规则。
- 支持任务进度管理：当前 TopOne 手动百分比、日常任务状态切换与排序；当长期任务进度达到 100% 时从当前 TopOne 释放并进入里程碑区，同时展示轻量完成庆祝。
- 支持将长期任务设为当前专注时的锁定时间选择，并在锁定期内提前切换时要求填写充分理由。
- 增加底部任务 / 奖励 / 设置导航，其中本 change 仅接通奖励与设置的占位页面，不实现奖励池正式管理、奖励抽取、完整完成归档成就页或 macOS 桌面常驻完整体验。

## Capabilities

### New Capabilities
- `task-page`: 覆盖任务页动态列表、任务 CRUD、进度管理、当前专注锁定与提前切换理由。

### Modified Capabilities
- 无

## Impact

- 影响 `Domain/Models/` 的长期任务与日常任务模型。
- 影响 `Persistence/Storage/PersistenceController.swift` 的 SwiftData schema。
- 影响 `Application/Services/` 的任务管理、进度管理与当前 TopOne 切换规则。
- 影响 `Presentation/` 与 `App/Shared/TopOneRootView.swift` 的任务页动态交互实现。
- 影响 `Tests/TopOneCoreTests/` 的领域规则与服务测试覆盖。
