# Design

记录技术方案目录索引，仅承载“系统如何实现”的设计入口，不再承载需求正文。

## 1. 目录职责

`docs/design/` 只存放技术方案，回答以下问题：
- 模块如何拆分
- 数据如何流转
- 接口如何定义
- 状态如何管理
- 存储与同步如何设计
- 有哪些技术权衡、风险与边界

以下内容不应写入本目录：
- 页面结构
- 页面跳转
- 交互流程
- 按钮文案
- 业务规则说明
- 纯文字原型描述

这类内容统一维护在 `docs/requirements/`。

## 2. 版本索引

| 版本 | 状态 | 说明 |
|------|------|------|
| [v1.0](./v1.0/README.md) | 当前 | 当前版本技术方案入口 |

## 3. 使用规则

- 每个产品版本在 `docs/design/<version>/README.md` 中维护独立技术方案。
- 根目录 `README.md` 只做索引，不重复抄写各版本正文。
- 技术方案应引用需求文档中的需求背景，而不是重复复制需求描述。
- 版本发布后，对应版本目录只读，不再持续混写新需求。

## 4. 与其他文档的关系

- 需求 / PRD：`docs/requirements/`
- 当前产品全量快照：`docs/product-snapshot/README.md`
- 高保真原型：`docs/prototype/README.md`
- 关键决策：`docs/decisions/`（按版本目录索引）
