# Prototype

记录高保真原型目录索引，仅承载可视化原型资产与版本入口，不再承载文字版需求说明。

## 1. 目录职责

`docs/prototype/` 只存放高保真原型与可视化交互证据，例如：
- Figma / 设计工具链接
- 页面截图
- 关键交互录屏
- 版本级页面清单

以下内容不应写入本目录：
- 纯文字页面地图
- 页面跳转说明
- 按钮文案与业务规则
- 需求背景与验收标准
- 技术实现说明

如果当前还没有高保真原型，相关页面与交互说明应继续维护在 `docs/requirements/`。

## 2. 当前状态

当前仓库已建立版本化原型目录骨架，但 `v1.0` 暂无正式高保真原型资产。

这意味着：
- 当前文字版页面与流程说明不再保留在本目录
- 在高保真原型产出前，以 `docs/requirements/` 作为页面与交互规则的唯一文字来源

## 3. 版本索引

| 版本 | 状态 | 说明 |
|------|------|------|
| [v1.0](./v1.0/README.md) | 待补齐资产 | 当前版本原型入口 |

## 4. 目录约定

每个版本目录建议包含：

```text
docs/prototype/
└── v1.0/
    ├── README.md          # 原型说明、页面清单、设计工具链接
    ├── screens/           # 页面截图
    └── flows/             # 关键交互录屏
```

- `screens/`：截图命名使用 `序号-页面名.png`
- `flows/`：录屏命名使用 `序号-流程名.gif`
- 版本发布后，对应版本目录只读
