# TopOne v1.0 原型交互流程

本文基于以下两类信息整理当前版本原型交互流程：
- Stitch 项目 `TopOne 专注任务提醒`
- 本地已落库页面 `docs/prototype/v1.0/screens/`

目标不是补新需求，而是把当前已有原型之间的跳转关系、触发动作和页面内关键交互写清楚，便于后续继续补图、验收和实现。

## 2. 总体导航关系

当前版本底部导航固定为 3 个入口：
- `任务`
- `奖励池`
- `设置`

跨页主导航关系：
- `任务列表页` ↔ `奖励池列表页`
- `任务列表页` ↔ `设置页`
- `奖励池列表页` ↔ `设置页`

说明：
- `iPadOS` 当前仅做尺寸适配，不单独提供另一套流程。
- 当前不包含 `macOS` 原型流程。

## 3. 核心流程

### 3.1 创建长期任务并开始专注

流程：
1. 用户在 `任务列表页` 点击右下角圆形 `+`
2. 触发动作菜单：`新增长期目标` / `新增日常任务`
3. 选择 `新增长期目标`
4. 进入 `创建长期任务输入页`
5. 用户填写长期任务内容后保存
6. 保存后返回 `任务列表页`
7. 用户在任务列表页选择某个长期任务作为当前专注目标
8. 触发 `专注选择 / 锁定交互`
9. 用户选择锁定周期后确认
10. 返回 `任务列表页`，该目标进入当前 `TopOne` 状态

补充说明：
- 当前版本不单独存在“创建后的承诺页”。
- 当前版本的“承诺 / 锁定”语义，放在任务列表页发起的专注选择交互里完成。

### 3.2 任务页日常使用流程

页面：`任务列表页`

关键交互：
- 点击当前 `TopOne` 进度百分比：手动编辑进度
- 拖动当前 `TopOne` 进度条：调整手动百分比
- 点击日常任务左侧圆形勾选框：切换完成状态
- 浏览日常任务列表：按 `执行中` → `未开始` → `已完成` 排序
- 查看暂存任务区：默认弱化 / 折叠，展开后只读
- 查看已达成里程碑区：默认折叠，展开后查看归档内容

说明：
- 无目标空状态不再单独成页，它归入“无任务时的任务列表页”。
- 当前原型中尚未单独提供“暂存任务展开态”和“已达成里程碑展开态”页面稿，但逻辑上属于任务页内部状态。

### 3.3 日常任务左滑操作流程

流程：
1. 用户在 `任务列表页` 或 `任务页左滑操作态` 中，对某条日常任务左滑
2. 显示操作按钮：`更新` / `编辑` / `删除`
3. 点击 `更新`
   - 进入任务进度调整交互
   - 保存后回到任务列表
4. 点击 `编辑`
   - 进入任务编辑页
   - 当前版本中“日常任务详情页”和“编辑页”视为同一页
5. 点击 `删除`
   - 触发二次确认
   - 确认后删除并回到任务列表

说明：
- 当前仓库中已有 `任务页左滑操作态`，用于表达左滑后的动作面板和删除确认。

### 3.4 提前更换专注目标流程

流程：
1. 用户在 `任务列表页` 尝试提前更换当前 `TopOne`
2. 进入 `强制更换页`
3. 用户输入更换理由
4. 页面实时显示理由字数差值
   - 未达标：负数，红色
   - 达标：显示 `0`，绿色
5. 用户可选择：
   - `再坚持一下`：返回 `任务列表页`
   - `我仍然要更换` / `确定放弃`：完成更换并返回 `任务列表页`

说明：
- 当前版本要求这个页面体现“有成本、有反思痕迹”，但不羞辱用户。

### 3.5 奖励池管理流程

页面：`奖励池列表页`

关键交互：
- 底部导航从 `任务列表页` 切换到 `奖励池列表页`
- 顶部视图切换：`列表模式` / `星辰占位模式`
- 当前正式管理入口为 `列表模式`
- 奖励项左滑：`编辑` / `删除`
- 删除奖励：需要二次确认
- 点击右下角圆形添加按钮：触发新增奖励交互

新增奖励交互说明：
- 当前版本不单独提供“新增奖励页”作为独立原型页
- 新增奖励交互并入 `奖励池列表页`
- 入口就是页面右下角的圆形 `+` 按钮
- 字段范围仍是：
  - 奖励内容
  - 奖励等级：`S / A / B / C`

说明：
- `星辰占位模式` 当前已有占位页原型，可用于表达氛围感与“敬请期待”状态。

### 3.6 完成后进入成就页流程

流程：
1. 用户完成长期目标
2. 系统生成完成结果并抽取奖励
3. 进入 `完成第二页 / 成就页`
4. 页面展示：
   - 完成感
   - 抽中的奖励
   - 奖励已永久存入对应长期目标的成就档案
5. 用户可进一步：
   - 立即领取奖励
   - 去奖励池查看更多
   - 返回日常使用路径

说明：
- 当前版本不再单独要求“完成第一页 / 回顾页”。
- 所以完成流程直接落到成就页表达最终结果。

### 3.7 设置页流程

页面：`设置页`

当前作用：
- 作为底部导航的一部分存在
- 当前以占位和基础设置表达为主
- 不承载本轮核心业务闭环

已体现的内容：
- 同步开关 / 存储状态
- 应用偏好
- 关于信息

说明：
- 设置页当前不是关键主流程页面，但原型中已存在，可用于后续补齐配置项。

## 4. 页面之间的跳转逻辑摘要

### 4.1 从任务列表页出发

- 点击 `+` → 选择 `新增长期目标` → `创建长期任务输入页`
- 点击 `+` → 选择 `新增日常任务` → 日常任务编辑 / 详情页（当前与编辑页合并）
- 选择某个长期任务为当前专注 → `专注选择 / 锁定交互`
- 提前更换当前专注 → `强制更换页`
- 底部导航切到 `奖励池` → `奖励池列表页`
- 底部导航切到 `设置` → `设置页`

### 4.2 从奖励池列表页出发

- 点击右下角圆形 `+` → 新增奖励交互
- 左滑奖励项 → 编辑 / 删除
- 切换到 `星辰占位模式` → `奖励池星辰占位页`
- 底部导航切到 `任务` → `任务列表页`
- 底部导航切到 `设置` → `设置页`

### 4.3 从成就页出发

- 点击 `立即领取奖励` → 奖励领取动作
- 点击 `在奖励池查看更多` → `奖励池列表页`
- 关闭或返回 → `任务列表页`

## 5. 当前流程闭环状态

当前版本用于主流程表达的页面已经基本齐备。

当前仍需后续继续增强的部分主要是：
- 星辰占位模式的视觉细节深化
- 更多任务页内部状态稿
- 更完整的流程图或录屏稿
