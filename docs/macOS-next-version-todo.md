# macOS 后续版本待做

本文记录当前版本暂不展开、但后续版本可继续评估的 `macOS` 设计方向。当前 `TopOne v1.0` 聚焦 `iOS` / `iPadOS` 主流程与高保真原型，`macOS` 不进入当前版本原型与实现范围。

## 1. 延后原因

- 当前高保真原型需要先覆盖 `iOS` / `iPadOS` 主流程。
- `macOS` 需要独立平台体验设计，不适合直接复用移动端页面。
- 当前版本优先验证“唯一 TopOne + 日常任务 + 奖励池 + 完成归档”的移动端闭环。

## 2. 已有设计线索

后续如重新启动 `macOS` 版本，可继续评估以下方向：

- 独立平台体验设计，而不是简单放大 `iOS` 页面。
- 桌面常驻展示层，用于让当前 `TopOne` 稳定可见。
- 状态栏入口，用于轻量查看当前状态或快速进入主窗口。
- 主窗口结构，用于承载完整任务、奖励与设置管理。
- 系统级触点需要始终服务“当前唯一注意力中心”，不能变成信息面板。

## 3. 后续需重新确认的问题

- `macOS` 是否仍属于 Apple-first 主线，还是作为独立版本阶段推进。
- 桌面常驻展示层的具体形态、尺寸、位置与关闭策略。
- 状态栏入口承载的信息边界。
- 主窗口与移动端信息架构之间的复用边界。
- `macOS` 与 `iOS` / `iPadOS` 之间的数据同步、冲突处理与离线策略。
- 是否需要单独的高保真原型、动效说明与平台验收标准。

## 4. 当前不做

- 不绘制 `macOS` 高保真原型。
- 不实现 `macOS` 页面或系统级展示触点。
- 不把 `macOS` 验收项纳入当前版本完成标准。
