## 1. 跨端设计原则
- 当前优先平台为 `iOS`、`iPadOS`
- `iPadOS` 基于 `iOS` 页面放大适配
- 各平台信息架构尽量一致
- 各平台交互方式遵守各自设备习惯

## 2. iOS / iPad 结构

### 2.1 总体结构
底部导航包含 3 个页签：
- `任务`
- `奖励池`
- `设置`

说明：
- `iPadOS` 沿用相同结构，基于 iOS 页面放大适配。
- 当前阶段不单独设计 iPad 独立信息架构。

### 2.2 任务页
页面定位：
- 任务页是当前 `TopOne` 的主要管理页面，必须始终突出当前唯一目标，而不是平铺所有目标。

页面结构：
1. 顶部导语（主标题 + 副标题）
2. 当前 `TopOne` 卡片
3. 当前 `TopOne` 的日常任务列表
4. 暂存任务区（只读冻结，按 `Rank S / A / B / C` 分组）
5. 已达成里程碑区（默认折叠）
6. 右下角圆形 `+` 浮动按钮

顶部导语：
- 使用主标题与副标题组合，强化当前唯一目标的精神性与推进感

当前 TopOne 卡片仅展示：
- 长期目标标题
- 手动百分比
- 锁定期剩余时间

明确不展示：
- 等级
- 阶段进度
- 其他管理信息

日常任务列表规则：
- 默认只展示当前 `TopOne` 的日常任务
- 日常任务默认按状态排序：
  1. `执行中`
  2. `未开始`
  3. `已完成`
- 点击左侧圆形勾选框可切换完成状态
- 左滑任务条提供：编辑、更新状态、删除

暂存任务区规则：
- 展示暂存任务
- 按 `Rank S / A / B / C` 分组
- 默认折叠或弱化显示，不抢占注意力
- 展开后为完整只读状态
- 可查看：长期目标标题、手动百分比、阶段信息、日常任务列表、任务状态、时间信息
- 不可编辑

已达成里程碑区规则：
- 展示已完成的长期目标
- 默认折叠
- 展开后可查看归档成果与完成结果摘要

任务页 `+` 按钮动作菜单：
- `新增长期目标`
- `新增日常任务`

日常任务详情页展示内容：
- 标题
- 状态
- 开始时间
- 结束时间
- 等级：`S / A / B / C`
- 对应奖励等级说明

日常任务详情页操作：
- 编辑
- 删除

### 2.3 奖励池页
页面定位：
- 奖励池页负责维护奖励内容，并通过视觉设计强化奖励的欲望感、期待感与惊喜感。

页面能力：
- 支持奖励的增删改查
- 支持两种展示模式：列表模式、星辰占位模式

列表模式：
- 按奖励等级优先级排序
- 默认分组顺序：`S` → `A` → `B` → `C`
- 是当前核心管理入口
- 奖励项左滑支持编辑、删除
- 删除必须二次确认

星辰占位模式：
- 当前为视觉占位模式
- 点击后提示敬请期待
- 不承载正式管理操作

新增奖励仅包含两个字段：
- 奖励内容
- 奖励等级：`S / A / B / C`

### 2.4 设置页
- 当前阶段暂时留空
- 不在本轮产品确认中展开设计

## 4. 创建与承诺流程

### 4.1 无目标状态首页
主文案：
- `现在请选择你最想做到的事情。`

小字文案：
- `你渴望，你获得。`
- `你看到，你成为。`
- `你反复，你成为。`

主按钮：
- `选择你的 TopOne`

### 4.2 首次创建目标输入页
标题：
- `写下那个在你心头出现过很多次的渴望`

辅助规则：
- 最大 16 字
- 提示语：`越短，越容易反复看见。`

按钮：
- `返回首页`
- `继续`

### 4.3 创建后的承诺页
主文案：
- `你愿意让它进入你接下来一段时间的注意力中心吗？`

按钮：
- `设为 TopOne`
- `先保留为候选`

### 4.4 锁定时间设置页
标题：
- `你愿意为它坚持多久？`

辅助文案：
- `一旦进入注意力中心，它就不该被轻易更换。`

选项：
- `7 天`
- `14 天`
- `30 天`
- `自定义`

按钮：
- `开始坚持`
- `返回`

## 5. 强制更换交互
页面作用：
- 不羞辱用户
- 但必须让更换变得有意识、有成本、有反思痕迹

文案结构：
- 标题：`你正在提前放下它。`
- 正文：
  - 真正改变你的，不是你想过多少次，
  - 而是你为同一个目标承受了多久。
  - 如果你仍然决定更换，请认真写下理由。
- 输入框提示：`为什么你现在必须更换它？`
- 按钮：`我仍然要更换`、`再坚持一下`

字数反馈设计：
- 使用差值形式显示，例如：`-50`、`-31`、`-12`、`0`
- 不加“还差”之类说明词
- 未达标时为红色
- 达标后显示 `0` 并变为绿色
- 每次字数变化时有跳动效果
- 最少字数旁有 `?` 图标，仅用于解释动态计算规则、设计用意、最大/最小边界

## 6. 完成交互

完成第一页：
- 沉静的回顾页
- 用于回顾和写成果总结
- 不提供返回按钮
- 主按钮自然推动进入第二页

完成第二页：
- 成就页
- 可被用户反复观看
- 页面体验目标包含：欢呼、快乐、长舒一口气、爽、欢乐、沉淀后的爆发感
- 展示随机抽取到的奖励
- 需要有品位和质感的背景

## 7. 首次进入与空状态

### 7.1 首次进入原则
- 首次进入产品时，只告诉用户产品可以做什么。
- 不强制用户立即创建目标。
- 不强制用户立即创建任务。
- 用户可以什么都不填写。

### 7.2 空状态引导
当用户尚未填写任何目标或任务时，界面可展示哲学性引导问题，例如：
- 你想成为什么样的人？
- 如果只能认真做好一件事，那会是什么？
- 什么事情值得你反复看见它？
