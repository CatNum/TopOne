## 1. 长期目标
- 长期目标是用户真正想做到的事情。
- 长期目标不是每日待办，也不是一次性任务。
- 长期目标分为：当前专注任务、暂存任务、已达成里程碑。
- 用户最多维护 3 个长期目标作为当前专注任务与暂存任务的总和。
- 同一时刻只能选择 1 个长期目标作为当前 `TopOne`。
- 暂存任务按 `Rank S / A / B / C` 分组展示。

## 2. TopOne
- `TopOne` 是当前唯一进入用户注意力中心的长期目标。
- 系统级展示位只展示当前 `TopOne`。
- `TopOne` 的标题由用户手动输入。
- 标题最少 1 个字，最多 16 个字。
- 推荐长度为 6 到 12 个中文字符。

## 3. 日常任务
- 每个长期目标都拥有自己的日常任务池。
- 日常任务是该长期目标的二级任务，用于推进长期目标。
- 只有当前 `TopOne` 的日常任务允许编辑。
- 暂存任务下的日常任务可以完整查看，但处于冻结只读状态。

## 4. 奖励池
- 奖励不在每个任务创建时单独设置。
- 用户提前维护一个奖励池。
- 奖励支持自由输入。
- 奖励字段仅包含：
  - 奖励内容
  - 奖励等级：`S` / `A` / `B` / `C`

## 5. 等级体系
- 长期目标、日常任务和奖励共用同一套等级体系：
  - `S`
  - `A`
  - `B`
  - `C`

等级含义：
- `S`：最高优先级 / 最高价值 / 最强激励，用于最重要的事项与奖励。
- `A`：高优先级 / 关键推进层，能显著推动当前目标进展。
- `B`：中优先级 / 稳定推进层，用于日常持续累积。
- `C`：基础优先级 / 轻量推进层，用于最低门槛的可执行行动与奖励。
