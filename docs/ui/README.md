# UI

> 模板提示：本文仅供参考，请结合项目实际进行人工完善与确认。

记录页面清单、组件规范、设计令牌和视觉约束。

## 1. UI 目标

TopOne 的 UI 不是为了展示“功能很多”，而是为了把用户的注意力稳定地收拢到当前唯一重要的那件事上。

因此 UI 设计必须同时满足：
- 极简
- 克制
- 有精神性
- 有情绪穿透力
- 有品位和质感
- 让用户感到“被一个目标持续看见”

## 2. 总体视觉原则

### 2.1 少就是多
- 页面同时强调的信息必须极少。
- 任何会削弱当前 `TopOne` 存在感的信息，都应弱化、折叠或延后展示。
- 不能把页面做成管理后台或待办工具面板。

### 2.2 主次分明
- 当前 `TopOne` 永远是第一注意力中心。
- 日常任务是推进层，不得与长期目标争夺视觉主位。
- 暂存任务必须可见，但默认弱化，不得喧宾夺主。

### 2.3 情绪不是装饰
- 文案、配色、留白、动效都要服务于产品哲学。
- 情绪表达应来源于“反复、承受、执行、练习、成为”，而不是靠堆砌夸张元素。
- 完成页与奖励页允许爆发感，但平时界面必须克制。

### 2.4 默认安静，接近才回应
- 常态界面尽量安静。
- 用户靠近、悬浮、点击时，再提供更多信息或操作。
- 奖励池的氛围型模式与强制更换等高情绪页面也应遵守“默认安静，交互后再回应”的原则。

## 3. 视觉气质

### 3.1 关键词
- 克制
- 安静
- 有重量
- 精神性
- 温和但坚定
- 干净
- 高级

### 3.2 避免的风格
- 传统待办软件感
- 高密度控制台感
- 炫技式科技感
- 过度可爱化
- 过度游戏化
- 低质 motivational slogan 风格

### 3.3 情绪分层
#### 日常状态
- 安静
- 稳定
- 有方向
- 不打扰

#### 任务推进状态
- 有轻微向前感
- 有节奏，但不焦躁

#### 强制更换状态
- 有压力感
- 有阵痛感
- 有自我面对感
- 不羞辱用户

#### 完成第一页
- 沉静
- 回顾
- 命名
- 收束

#### 完成第二页
- 欢呼
- 快乐
- 长舒一口气
- 爽
- 欢乐
- 沉淀后的爆发

## 4. 页面层级规则

### 4.1 首页 / 无目标状态
视觉重心：
1. 主文案
2. 小字文案
3. 主按钮

要求：
- 留白充足
- 不堆信息
- 主按钮有明确存在感，但不过于商业化
- 文案区有思考感，不像 onboarding 广告页

### 4.2 任务页
视觉重心：
1. 当前 `TopOne`
2. 日常任务列表
3. 暂存任务分组区
4. `+` 浮动按钮

要求：
- 顶部导语必须作为任务页第一段内容，使用主标题 + 副标题表达当前唯一目标的精神引导
- 当前 `TopOne` 必须是页面第一视觉锚点
- 暂存任务必须明显弱于当前 `TopOne`
- 已达成里程碑必须再弱于暂存任务，呈现归档感而不是成就徽章感
- 日常任务列表保持清晰节奏感
- 长期任务条目不使用图标

### 4.3 奖励池页
视觉重心：
1. 奖励内容本身
2. 等级分层
3. 模式切换
4. `+` 浮动按钮

要求：
- 奖励池页允许比任务页更有审美表达
- 但仍要保持高级感，不走幼稚化路线

## 5. 文案层级规范

### 5.1 主文案
- 用于承载产品哲学或关键引导
- 句子短，但要有力量
- 不能像功能说明
- 不能像广告口号

### 5.2 辅文案
- 用于补足哲学意图与情绪氛围
- 不能喧宾夺主
- 通常使用更小字号、更弱颜色

### 5.3 操作文案
- 按钮文案要直接、清晰、有态度
- 避免技术术语
- 避免过长

### 5.4 状态提示
- 例如字数不足、锁定期、冻结状态等
- 应保持冷静，不嘲讽，不训诫
- 强调“这是一次认真选择”，而不是“你做错了”

## 6. 色彩原则

### 6.1 基础方向
TopOne 的日常界面应以中性色为主，通过极少量重点色完成引导。

建议色彩结构：
- 主体背景：柔和浅色或低刺激中性色
- 内容卡片：更干净的浅色容器
- 主文本：高对比深色
- 次文本：中低对比灰色
- 强调色：少量使用，用于主按钮、关键状态、焦点反馈

### 6.2 状态色建议
- 正常推进：克制的主色
- 达成 / 完成：偏明亮但不廉价的成功色
- 字数不足 / 阵痛提醒：偏红，但不刺眼
- 达标切换：偏绿，用于从负数变到 `0` 的那一刻

### 6.3 奖励页色彩
- 可以更有情绪张力
- 允许更高饱和度和更强对比
- 但必须保留高级感和质感，不能廉价热闹

## 7. 字体与排版

### 7.1 字体原则
- 优先使用系统字体
- 需要自然、稳定、可长期阅读
- 不追求装饰性字体

### 7.2 标题层级
- 首页主文案：最大字号层级之一
- 当前 `TopOne` 标题：任务页最高层级文本
- 页面标题：清楚但不过于抢眼
- 辅助说明：小一到两级

### 7.3 行高
- 主文案行高可略紧，增强力量感
- 列表内容行高应舒适，增强可扫读性
- 小字文案保持轻盈感

### 7.4 数字展示
- 百分比、倒计时、差值数字都属于重要信息
- 差值数字如 `-50` 到 `0` 应具备足够视觉存在感
- 数字切换时动效应短促、有弹性，不拖沓

## 8. 留白与密度

### 8.1 留白原则
- 重要页面必须敢于留白
- 留白本身就是一种强调方式
- 空状态页面尤其不能填满

### 8.2 列表密度
- 日常任务列表要易于快速扫读
- 单项高度不宜过高，也不宜过密
- 完成项应明显弱化，但仍保留可回顾性

### 8.3 卡片密度
- 当前 `TopOne` 卡片要有呼吸感
- 暂存任务分组区可以更紧凑
- 奖励池列表卡片允许更轻盈、更有趣味性

## 9. 核心组件规则

### 9.1 当前 TopOne 卡片
展示：
- 长期目标标题
- 手动百分比
- 剩余锁定时间倒计时

规则：
- 不显示等级
- 不显示阶段
- 不显示杂项操作
- 卡片应像“注意力核心”，不是信息面板

### 9.2 日常任务项
展示：
- 标题
- 状态
- 必要时间信息

规则：
- `执行中` 必须最容易被看见
- `未开始` 保持清晰但更轻
- `已完成` 明显弱化
- 不应出现多余标签和附加元信息

### 9.3 暂存任务分组区
规则：
- 默认折叠或弱化
- 按 `Rank S / A / B / C` 分组
- 展开后完整只读
- 明确呈现“存在但冻结”的状态
- 禁止比当前 `TopOne` 更强势
- 弱化必须通过更低对比背景、更轻文案、更少装饰与更紧凑容器共同实现，不能只靠轻微透明度
- 长期任务条目不显示图标

### 9.4 奖励项
列表模式：
- 更强调文字内容与等级
- 排序清晰

星辰占位模式：
- 当前为视觉占位模式
- 更强调氛围感与期待感
- 点击后提示敬请期待

### 9.5 `+` 浮动按钮
适用页面：
- 任务页
- 奖励池页

规则：
- 永远位于右下角
- 保持足够可点击性
- 形态统一为圆形
- 阴影和颜色要有存在感，但不粗暴抢眼
- 任务页点击后直接进入统一的创建任务页面，不先弹动作菜单
- 创建页内部用分段切换承载 `日常任务` / `长期任务`
- 创建页底部保留弱化名言区，帮助页面在操作区之后自然收束

### 9.6 差值数字组件
适用场景：
- 强制更换理由输入页

规则：
- 初始显示为负数，如 `-50`
- 未达标：红色
- 达标：绿色 `0`
- 每次输入变化时触发跳动效果
- `?` 图标仅解释规则与用意，不作其他说明

## 10. 动效规则

### 10.1 总原则
- 动效应短、准、克制
- 不做花哨转场
- 不做为了“炫”而存在的动效

### 10.2 日常动效
- 按钮点击：轻微缩放或明暗反馈
- 列表展开收起：平滑但迅速
- 状态切换：清晰、无拖泥带水

### 10.3 关键动效
- 差值数字：跳动反馈
- 奖励抽取：应有“揭晓感”与“期待感”
- 完成第二页：允许更强烈的情绪释放动效

## 11. 设计令牌使用方向

当前已有 `docs/ui/tokens/design-tokens.json`，后续应按以下方向收紧：

### 11.1 色彩
- 当前主色仍是占位值，后续需替换为更符合 TopOne 气质的主色体系
- 中性色需要重点打磨，因为产品大量依赖留白、层级与文字力量
- 成功、错误、警示色需确保高级而非廉价

### 11.2 阴影
- 阴影应服务于层级，而不是制造夸张悬浮感
- 奖励页与完成页会更依赖阴影质感

### 11.3 圆角
- 保持现代但不过于圆润
- 奖励池与浮动按钮可略圆
- 信息卡片与操作组件圆角需统一

### 11.4 字号
- 当前 token 可作为初始基线
- 后续需围绕主文案、TopOne 标题、数字信息做更精细的层级划分

## 12. 后续 UI 深化建议

建议下一步按以下顺序推进：

1. 先确定首页、任务页、奖励池页的低保真线框结构
2. 再确定当前 `TopOne` 卡片、日常任务项、暂存任务分组区三类核心组件
3. 再打磨完成第二页与奖励页背景风格
4. 最后回收并更新 `design-tokens.json`
